2. วัตถุประสงค์เชิงธุรกิจ

ระบบโปรโมชั่นเป็นเครื่องมือของฝ่ายการตลาดในการกระตุ้นยอดขายและรักษาลูกค้า ครอบคลุมส่วนลดต่อสินค้า / ตะกร้ารวม / ของแถม / คูปอง / โปรโมชั่นตามระดับสมาชิก โดยคำนวณอัตโนมัติในขั้นตอนชำระเงินทุกช่องทาง

รหัสเป้าหมายธุรกิจตัวชี้วัด
BG-PROMO-1ฝ่ายการตลาดสร้างแคมเปญได้เองโดยไม่ต้องพึ่ง ITเวลาในการสร้างแคมเปญใหม่ ≤ 15 นาที
BG-PROMO-2ระบบคำนวณส่วนลดอัตโนมัติทุกช่องทาง (เว็บ / CC / POS)ความตรงกันของการคำนวณ 100%
BG-PROMO-3รองรับโปรโมชั่นหลายแบบในเวลาเดียวกันรันพร้อมกันได้ ≥ 50 แคมเปญ
BG-PROMO-4วัดผลและรายงานประสิทธิภาพแต่ละแคมเปญมีรายงานครบทุกแคมเปญที่จบ

3. ขอบเขตของงาน (In / Out of Scope)

3.1 อยู่ในขอบเขต (In Scope)

  • การสร้างแคมเปญส่วนลด: %, ค่าคงที่, ส่วนลดตามจำนวน (Tier)
  • การสร้างคูปอง: รหัส, จำนวนครั้งใช้, วันหมดอายุ, ลูกค้าที่กำหนด
  • การสร้างโปรโมชั่นของแถม (Buy X Get Y)
  • การสร้างโปรโมชั่นตามระดับสมาชิก (Prospect ถึง Coronet)
  • การคำนวณส่วนลดอัตโนมัติในขั้นตอนชำระเงิน
  • การแสดงป้าย "Promotion" หน้าเว็บไซต์ (เชื่อม PRD-11)
  • การยกเลิกหรือหยุดแคมเปญชั่วคราว
  • รายงานประสิทธิภาพแคมเปญ

3.2 ไม่อยู่ในขอบเขต (Out of Scope)

  • Pre-Order ไม่รองรับโปรโมชั่นใดๆ (ต้องชำระ 100%)
  • การจัดการแคมเปญในระดับเครือข่ายร้าน ดูเอกสาร CMS BRD
  • การกำหนดราคาขั้นพื้นฐานของสินค้า ดูเอกสาร PRD BRD

4. ผู้มีส่วนได้ส่วนเสียและบทบาทผู้ใช้งาน

บทบาทหน้าที่สิทธิ์
ผู้ดูแลโปรโมชั่น (Promo Admin)สร้าง / แก้ไข / ยกเลิกแคมเปญเต็มสิทธิ์
หัวหน้าฝ่ายการตลาดอนุมัติแคมเปญใหม่ก่อนเปิดใช้อนุมัติ
ฝ่ายการเงินตรวจสอบผลกระทบทางการเงินของแคมเปญดู + ตรวจ
ลูกค้าใช้คูปองและรับโปรโมชั่นในขั้นตอนชำระเงินใช้
พนักงาน CC / POSแจ้งโปรโมชั่นและช่วยใช้คูปองให้ลูกค้าใช้ (แทนลูกค้า)
ระบบ ConX ERPรับยอดส่วนลดสำหรับการคำนวณบัญชีAPI

5. สถานะปัจจุบัน (As-Is)

  • แคมเปญใช้กระดาษและ Excel ฝ่ายการตลาดเขียนกฎและส่งให้ IT แปลงเป็นโค้ด
  • คูปองพิมพ์เป็นกระดาษ ลูกค้าเอามาให้แคชเชียร์
  • ไม่มีการคำนวณอัตโนมัติ แคชเชียร์ต้องคำนวณส่วนลดเอง
  • ไม่มีรายงานประสิทธิภาพ ฝ่ายการตลาดเดาผลโดยดูยอดขายช่วงแคมเปญ

6. กระบวนการในอนาคต (To-Be)

  • Promotion Engine ที่ตั้งค่าได้ ฝ่ายการตลาดสร้างเองได้
  • คูปองเป็นรหัสดิจิทัล ส่งทางอีเมล / LINE
  • คำนวณส่วนลดอัตโนมัติ เห็นทันทีในตะกร้า
  • รายงานประสิทธิภาพ ยอดขาย / จำนวนลูกค้า / Cost of Discount

13. ข้อกำหนดเชิงคุณภาพ (NFR)

หัวข้อเกณฑ์
เวลาตอบสนองคำนวณส่วนลดในตะกร้า ≤ 1 วินาที
การรองรับรันได้ ≥ 50 แคมเปญพร้อมกัน
ความถูกต้องผลคำนวณตรงกัน 100% ระหว่างทุกช่องทาง

14. รายงานและแดชบอร์ดที่เกี่ยวข้อง

  • รายงานยอดขายจากแคมเปญ ตามแคมเปญ / ช่องทาง / สินค้า
  • รายงานการใช้คูปอง ใช้กี่ครั้ง โดยใคร
  • รายงาน Cost of Discount ส่วนลดรวมเทียบกับยอดขาย
  • แดชบอร์ดสำหรับฝ่ายการตลาด แคมเปญที่กำลังจะหมดอายุ / ใกล้เริ่ม

15. ข้อกำหนดการโอนย้ายข้อมูล (Migration)

  • แคมเปญใหม่เริ่มในระบบใหม่ ไม่โอนแคมเปญเก่า
  • คูปองค้างจากระบบเก่าให้ลูกค้าใช้ผ่านการป้อนรหัสด้วยมือ จนกว่าหมดอายุ

16. สมมติฐานและข้อจำกัด

  • การคำนวณส่วนลดในระบบใหม่จะเป็นแหล่งอ้างอิงเดียว
  • ConX รับยอดส่วนลดและบันทึกในบัญชี
  • Pre-Order ไม่รองรับโปรโมชั่นใดๆ
