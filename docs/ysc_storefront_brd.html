เอกสารฉบับนี้ระบุข้อกำหนดของเว็บไซต์สำหรับลูกค้า (Storefront) ของยงเจริญศูนย์เครื่องเขียน ครอบคลุมหน้าแรก หน้ารายการสินค้า หน้ารายละเอียดสินค้า ตะกร้า กระบวนการชำระเงิน 4 ขั้นตอน การเข้าสู่ระบบ/สมัครสมาชิก และการจัดการบัญชี โดยอ้างอิงMockup จาก Figmaที่แนบมาในเอกสารและกฎเกณฑ์เชิงธุรกิจจาก ysc_business_rules.html v2.2

หมายเหตุเกี่ยวกับ Figma: ดีไซน์ในไฟล์ Figma ปัจจุบันครอบคลุมกระบวนการซื้อสินค้าหลักเท่านั้น (Homepage, Product Detail, Cart, Checkout, Login) ดีไซน์ที่ยังไม่มีในไฟล์ Figmaและต้องสร้างเพิ่ม ได้แก่ Registration (General/Dealer), OTP, My Account Dashboard, Order History, Quotation Request, Pre-Order, Address Book, Loyalty Points รายการเหล่านี้ระบุไว้ในแต่ละ Use Case พร้อม รอออกแบบเพิ่ม

2. วัตถุประสงค์เชิงธุรกิจ

เว็บไซต์สำหรับลูกค้าเป็นช่องทางการขายออนไลน์หลักของยงเจริญฯ ที่ลูกค้าสามารถสมัครสมาชิก ค้นหาและสั่งซื้อสินค้า จัดการคำสั่งซื้อและบัญชีของตนเองได้ โดยรองรับทั้งลูกค้าบุคคลธรรมดาและลูกค้านิติบุคคล (ที่มีวงเงินเครดิตและประเภทใบกำกับภาษีพิเศษ)

รหัสเป้าหมายธุรกิจตัวชี้วัด (KPI)
BG-WEB-1เพิ่มยอดขายผ่านช่องทางออนไลน์ ลดภาระของ Contact Centerสัดส่วนคำสั่งซื้อจากเว็บ ≥ 40% ในปีที่ 1
BG-WEB-2ลูกค้าสมัครและสั่งซื้อได้สะดวกในขั้นตอนน้อยอัตราการสมัครสำเร็จ ≥ 90%, Cart Abandonment Rate ≤ 60%
BG-WEB-3รองรับทั้งลูกค้าบุคคลธรรมดาและนิติบุคคลในเว็บไซต์เดียวรองรับการขอวงเงินเครดิตและใบกำกับภาษี Y/N/M/A ครบ
BG-WEB-4ดีไซน์เว็บไซต์สวยและใช้งานง่ายบน Desktop และ MobileMobile Friendly Score ≥ 90/100, Web Vitals ผ่านเกณฑ์ Google
BG-WEB-5ค้นหาและกรองสินค้าได้รวดเร็วค้นหาแสดงผลใน ≤ 1 วินาที, Auto-suggest ระหว่างพิมพ์

3. ขอบเขตของงาน

3.1 อยู่ในขอบเขต (In Scope)

  • หน้าแรก (Homepage) — Banner, สินค้ามาใหม่, โปรโมชั่น, แบรนด์ ฯลฯ
  • การค้นหาสินค้า — Search Bar พร้อม Auto-suggest
  • หน้ารายการสินค้าตามหมวด (Product Listing)
  • หน้ารายละเอียดสินค้า (Product Detail Page — PDP)
  • ตะกร้าสินค้า (Cart) และกระบวนการชำระเงิน 4 ขั้นตอน:
    1. ตรวจสอบตะกร้า
    2. ที่อยู่และวิธีส่ง
    3. เลือกวิธีการชำระเงิน
    4. ตรวจสอบและยืนยัน
  • หน้า Login / Register สำหรับลูกค้าบุคคลธรรมดาและนิติบุคคล
  • OTP Verification ตาม AUTH-02, NTF-03
  • My Account — ดูประวัติคำสั่งซื้อ คะแนนสะสม สมุดที่อยู่ ใบเสนอราคา
  • การขอใบเสนอราคา (Quotation Request) ตาม Q-01 ถึง Q-04
  • การสั่งซื้อล่วงหน้า (Pre-Order)
  • Responsive Web Design รองรับ Desktop / Tablet / Mobile
  • SEO Basics — Sitemap, Meta tags, Schema.org markup

3.2 ไม่อยู่ในขอบเขต (Out of Scope)

  • Mobile Native App (iOS/Android) — ใช้ Responsive Web ไปก่อน
  • ระบบสมาชิกแบบโซเชียล (Login ด้วย Facebook / Google) — เฟสถัดไป
  • การแชทสด (Live Chat) ใช้ LINE OA แทน
  • ระบบรีวิวสินค้า / ดาวคะแนน — เฟสถัดไป

4. ผู้มีส่วนได้ส่วนเสียและบทบาทผู้ใช้งาน

บทบาทหน้าที่ในเว็บไซต์สิทธิ์
ลูกค้าทั่วไป (Guest)ดูสินค้า อ่านโปรโมชั่น แต่สั่งซื้อไม่ได้ ตาม AUTH-01ดูอย่างเดียว
ลูกค้าบุคคลธรรมดา (สมาชิก)สั่งซื้อ ชำระเงิน ดูประวัติ จัดการที่อยู่ คะแนนเฉพาะข้อมูลตนเอง
ลูกค้านิติบุคคลสั่งซื้อด้วยวงเงินเครดิต ขอใบกำกับ Y/N/M/A ขอใบเสนอราคาเฉพาะข้อมูลบริษัทตนเอง
พนักงาน CCใช้เว็บไซต์ในการสาธิตและตอบคำถามลูกค้าดูอย่างเดียว

5. โครงสร้างข้อมูลและสถาปัตยกรรม (Sitemap)

เว็บไซต์มีโครงสร้างหลัก 5 ส่วน ตามแผนภาพต่อไปนี้ (ภาพจาก Figma Sitemap):

Sitemap ภาพรวมของเว็บไซต์ — 5 กลุ่มหลัก: สินค้า, โปรโมชั่น, วิธีสั่งซื้อ, เกี่ยวกับเรา, บัญชีของฉัน Figma node 437:1248
ส่วนหน้าจอย่อย
หน้าหลักHomepage, Search Results, สินค้ามาใหม่, สินค้าขายดี, โปรโมชั่นปัจจุบัน
สินค้าหมวดหมู่สินค้า (Category), รายการสินค้า (Listing), รายละเอียดสินค้า (PDP), สินค้าที่คล้ายกัน, สินค้าทดแทน
โปรโมชั่นแคมเปญที่กำลังรัน, สินค้าตามแคมเปญ, คูปองของฉัน
ตะกร้าและชำระเงินตะกร้า, Checkout 4 ขั้นตอน, ใบเสนอราคา, สั่งซื้อสำเร็จ
บัญชีของฉันโปรไฟล์, ประวัติคำสั่งซื้อ, สมุดที่อยู่, คะแนนสะสม, ใบเสนอราคา, Pre-Order, ขอวงเงินเครดิต
ข้อมูลทั่วไปเกี่ยวกับเรา, วิธีการสั่งซื้อ, นโยบายความเป็นส่วนตัว, คำถามที่พบบ่อย (FAQ), ติดต่อเรา

5.1 Flow ภาพรวม (E2E)

Flow End-to-End ของกระบวนการสั่งซื้อ ตั้งแต่ค้นหาสินค้าจนถึงรับสินค้า Figma page Flow E2E

6. สถานะปัจจุบัน (As-Is) และกระบวนการในอนาคต (To-Be)

6.1 สถานะปัจจุบัน (As-Is)

  • เว็บไซต์เดิมมีลักษณะเป็น Catalog-only ลูกค้าโทรสั่งหรือมาที่ร้านเอง
  • ไม่มีระบบสมัครสมาชิก ไม่มีตะกร้า ไม่มี Checkout
  • ลูกค้านิติบุคคลใช้ Email และ Phone Call ในการสั่งซื้อ

6.2 กระบวนการในอนาคต (To-Be)

  • เว็บไซต์ E-Commerce เต็มรูปแบบ ลูกค้าสามารถสั่งซื้อและชำระเงินครบในเว็บ
  • ลูกค้านิติบุคคลสมัครและขอวงเงินได้จากเว็บ ไม่ต้องส่งเอกสารทาง Email
  • เชื่อมต่อ Backend (OMS) และ ConX ERP แบบเวลาจริง
  • รองรับ Responsive Web ทุกขนาดหน้าจอ

  2. ระบบแสดงAuto-suggest Dropdownทันที (ตั้งแต่พิมพ์ 2 ตัวอักษร) ประกอบด้วย:
    • หมวดหมู่ที่ตรง
    • สินค้าแนะนำ 5 รายการแรก (รูป, ชื่อ, ราคา)

  4. ระบบเปิดหน้าผลการค้นหา แสดงรายการทั้งหมดพร้อมตัวกรอง (แบรนด์, ราคา, หมวด)

1a. ลูกค้าค้นด้วยSKU ระบบไปที่หน้ารายละเอียดสินค้านั้นทันที
2a. ไม่พบสินค้า ระบบเสนอ "ลองค้นหาคำอื่น" + สินค้ายอดนิยม

  2. ระบบเปิด หน้ารายละเอียดสินค้า แสดง:

    • ราคา (แสดงตามระดับราคาของลูกค้า P1-P5 ที่ Login)
    • สต็อกคงเหลือ (มี / ใกล้หมด / Pre-Order)
    • ตัวเลือกจำนวน และปุ่ม "เพิ่มลงตะกร้า"

  2. ไอคอนตะกร้าใน Header แสดงจำนวนรายการที่อัปเดต
  3. ลูกค้าคลิกไอคอนตะกร้า ระบบเปิด หน้าตะกร้า
  4. หน้าตะกร้าแสดง:
    • Breadcrumb Step 4 ขั้นตอน (ขั้นปัจจุบัน = 1 ตรวจสอบตะกร้า)
    • 3 Tabs: รายการสั่งซื้อ / รายการขอใบเสนอราคา / รายการ Pre-Order
    • รายการสินค้าจัดกลุ่มตามแบรนด์ (ตามภาพ Figma)
    • แต่ละรายการ: รูป, ชื่อ, SKU, จำนวน (ปุ่ม +/-), ราคา/ชิ้น, ราคารวม, ปุ่มลบ
    • โปรโมชั่นที่ยังไม่ถึงเงื่อนไข (เช่น "ซื้ออีก ฿280 บาทรับส่วนลด ฿250")
    • Right Panel: เลือกประเภทการรับสินค้า (รับเอง / ส่ง), สรุปยอด, ค่าจัดส่ง, ปุ่ม "ดำเนินการต่อ"
  5. ลูกค้าปรับจำนวน ระบบคำนวณราคาใหม่ทันที
  6. ลูกค้ากด "ดำเนินการต่อ" ไป Step 2

ลูกค้าระบุที่อยู่จัดส่ง ที่อยู่ออกใบกำกับ และเลือกวิธีจัดส่ง

ขั้นตอนหลัก
  1. ระบบแสดงที่อยู่จัดส่งของลูกค้า (ถ้ามีในสมุดที่อยู่ ให้เลือก / กรณีไม่มี ให้เพิ่มใหม่)
  2. ลูกค้าเลือกหรือเพิ่มที่อยู่
  3. ลูกค้าเลือกประเภทใบกำกับภาษี (Y / N / M / A ตาม INV-04)
  4. ลูกค้าเลือกวิธีการจัดส่ง:
    • จัดส่งโดย YSC (สาย A/B/C/ทั่วประเทศ) ตาม SHP-02
    • จัดส่งด่วนภายในวัน (Express + ค่าธรรมเนียม) ตาม SHP-05
    • จัดส่งโดยขนส่งเอกชน (200+ ราย Carrier ตามเขต) ตาม SHP-03
    • รับเอง (Self-Pickup) ตาม SHP-08, SHP-09
  5. กรณี Self-Pickup ลูกค้าเลือกวันเวลานัด (≥ 12 ชั่วโมงจากตอนสั่ง)
  6. ลูกค้ากด "ดำเนินการต่อ" ไป Step 3

12. ความปลอดภัยและความเป็นส่วนตัว (Security & Privacy)

  • HTTPS ทุกหน้าเว็บไซต์ (SSL Certificate)
  • การยืนยันตัวตน ใช้ Session-based + JWT, รองรับ Remember Me 30 วัน
  • OTP สำหรับการสมัคร / เปลี่ยนรหัส / ธุรกรรมเสี่ยง ตาม NTF-03, NTF-04
  • การชำระเงินด้วยบัตร ส่งต่อให้ Payment Gateway ตามมาตรฐาน PCI-DSS — ไม่เก็บข้อมูลบัตรในระบบ
  • PDPA ขอความยินยอมเก็บและประมวลผลข้อมูลส่วนตัว
  • Rate Limiting ป้องกัน Brute-force Login
  • CAPTCHA ในหน้าสมัครสมาชิก และหลัง Login ผิด 3 ครั้ง
