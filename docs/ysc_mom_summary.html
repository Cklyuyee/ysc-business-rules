2 MoM 8 มกราคม 2569 — การประชุมเริ่มต้นโครงการ

8 ม.ค. 2569 — Kickoff Meeting

ผู้เข้าร่วม: Interblend (UI/UX) + Adeptio (Backend) + ทีมยงเจริญ

วัตถุประสงค์: กำหนด Scope ของเว็บไซต์ (Frontend / Backend / CMS) และ Flow การเชื่อมต่อ CONX

ข้อมูลสำคัญที่บันทึก

  • หน้าแรก (Homepage): 3 Banners ปรับเปลี่ยนได้ — Section "Best Deals of the Week" + หมวดสินค้าใหม่ + หมวดโปรโมชั่น
  • ไอคอนสินค้า 2 ประเภท: Promotion + Green Product
  • Pre-order 2 สถานะ: สั่งซื้อได้ และ ยังไม่สามารถสั่งซื้อได้ (ต้อง Verify)
  • หน้ารายละเอียดสินค้า: ตัวเลือกสูงสุด 3 ค่า (สี/ไซส์/Attribute) + Barcode 13 หลัก
  • Threshold ขอใบเสนอราคา: ยอด ≥ 5,000 ชิ้น หรือ 50,000 บาท → แสดงปุ่ม "ขอใบเสนอราคา"
  • การสมัครสมาชิก B2C: อีเมล / เบอร์โทร + OTP ทุกครั้ง + รองรับ Social Login (LINE / Gmail)
  • การสมัครสมาชิก B2B: จดทะเบียน / ไม่จดทะเบียน + บังคับเบอร์โทร + OTP ทุกครั้ง
  • CMS Structure: Content (Pages, Articles/SEO, Media, FAQ), Report, Customer (Customer, Suppliers), Solution B2B (Business Type, Product Mapping)
  • B2B Lead: เก็บ Lead ไม่ส่ง CONX
  • 2 Warehouses ใน CONX: WH สำหรับ Online + Agent / WH สำหรับช่องทางหน้าร้าน

3 MoM 14 มกราคม 2569 — End-to-End Process

14 ม.ค. 2569 — Decision Records D-001 ถึง D-005

ผู้เข้าร่วม: พี่โอ๊ะ (จัดซื้อ), พี่รัตน์ (WH Mgr), พี่เฟิร์ส (รอง WH Mgr), พี่กุ้ง (CC), พี่พร (บัญชี), Adeptio

วัตถุประสงค์: สรุปภาพรวม E2E Process — Product → Order → Stock → Payment → Accounting

ประเด็นสำคัญ

  • ประเภทสินค้า 14 ประเภท: Normal, Pre-order, Free, Quotation, New, Active/Inactive, Promotion, CC only, POS only, ฝากขาย, จ้างผลิต, E-Commerce, สินค้าทดแทน, สินค้าใกล้เคียง
  • Pre-order Logic: ไม่มี Stock / มีราคา Pre# / ไม่มีราคา (ติดต่อ CC) / เงื่อนไขมัดจำ 50%, 100% หรือไม่ต้องมัดจำ / ลูกค้าเครดิตอาจไม่ต้องมัดจำ
  • Barcode & SKU: 1 สินค้า = 1 SKU (หลาย Barcode ได้) + Barcode ผูกตามหน่วยขาย + เก็บ Barcode เก่า + Log การเปลี่ยนแปลง
  • Warehouse Operation: สีสถานะ — ขาว (ครบ), เหลือง (ขาด), ฟ้า (ยังไม่จัด), แดง (ไม่ตรง)
  • Stock Movement: รับเข้า / โอน / Pick-Pack / Stock ตัดเมื่อ Invoice / On-hand & Available Update Real-time
  • Order Status Flow:
    • Cash: Create → Confirm → Pick/Pack → Invoice → Paid → Delivery → Close
    • Credit: Create → Confirm → Pick/Pack → Invoice → Credit → Delivery → Paid → Close
  • VAT/Invoice Logic 6 รูปแบบ: Y/N/M/A combinations กับ V/I → Tax Invoice variations
  • Payment Types: เงินสด, เครดิต, กทม., ตจว.
  • CC Functions: Print ใบสั่งซื้อ (สีเขียว = ด่วน 1-2 ชม.)

คำถามจากพี่หมวย

  • คลัง 4 = Web + Agent (ยืนยัน)
  • Payment Type ครบทุกประเภทหรือยัง?
  • Pre-order มัดจำทุกกรณี?
  • Flow ที่ถูกต้อง: SO > pick > pack > ใบส่งสินค้า > ชำระ > ใบกำกับ > ใบเสร็จ > ส่งสินค้า > ปิด

4 MoM 16 มกราคม 2569 — CC Process & Agent Portal

16 ม.ค. 2569 — Decision Records D-006 ถึง D-009

หัวข้อ: CC Process & Order Flow (Sales Order Process)

ทบทวนกระบวนการ Call Center ตั้งแต่ลูกค้าติดต่อจนถึงการสร้างคำสั่งซื้อ

การออกแบบ CC Process

  • ช่องทางที่ลูกค้าติดต่อ: โทรศัพท์ / LINE / อีเมล / โทรสาร
  • เมื่อมีการติดต่อ ระบบต้อง: เปิด Ticket → ดึงข้อมูลลูกค้า → แสดงประวัติการซื้อ → แสดงสถานะลูกค้า (Credit, ค้างชำระ, Tier)

Customer Profile (CRM Information)

  • รหัสลูกค้า / ชื่อร้าน (TH/EN) / ที่อยู่จัดส่ง / ที่อยู่ออกใบกำกับ / ชื่อผู้ติดต่อ + เบอร์โทร / ประเภทธุรกิจ / Customer Tier
  • ประเภทการชำระเงิน (เงินสด / เครดิต — จำนวนวัน + วงเงิน)
  • Credit Limit + วงเงินคงเหลือ Real-time
  • ระบบแจ้งเตือนทันที หากวงเงินเครดิตไม่เพียงพอ

Order History & Frequent Items

  • แสดงยอดซื้อเฉลี่ยย้อนหลัง: 1 / 3 / 6 / 12 เดือน
  • Order ล่าสุด = Submit แล้ว (ไม่รวม Cancelled)
  • Top 10 Items ใน 6 เดือน — SKU + Barcode + จำนวนเฉลี่ย/ครั้ง + จำนวนครั้งที่ซื้อ
  • ใช้เพื่อ: เสนอโปรโมชั่นที่เหมาะสม + แจ้งเตือนสินค้าที่ใกล้หมด

Shelf Template (Wishlist) NEW

  • วิธีสร้าง Order 3 แบบ: คีย์ใหม่ / ดึงจาก Order History / ใช้ Template
  • Template ที่ลูกค้าสร้าง: จำกัด 3 Template ต่อ Account
  • Template ที่ทีม Agent/Marketing สร้าง 4 ประเภท:
    • สำหรับทุกคน (mass)
    • สำหรับ tier ต่างๆ
    • สำหรับ category ตามประเภทธุรกิจ
    • สำหรับรายบุคคล (ตามรหัสสินค้า / กลุ่มลูกค้า)
  • มี Active / Inactive + Start / End Date
  • Auto Inactive เมื่อหมดอายุ
  • Mass Template แสดงให้ลูกค้าใหม่ที่ยังไม่ login ได้

Agent Portal Menus

  • Sales Order: Save Draft ได้ + ห้ามเปิด SO ถ้า Stock Online ติดลบ
  • Customer Information: ตรวจสอบข้อมูลลูกค้า
  • Product Information: ค้นหาด้วย รหัส / Barcode / ชื่อสินค้า (มี Suggest Keyword) + แสดง Location, Stock Online/Offline, ราคา Last Order 6 เดือน
  • Promotion: แสดงโปรโมชั่น
  • Ticketing: Track การทำงานของ Agent

Credit & Permission

  • Agent เห็น Credit Limit ตั้งแต่ต้น
  • เกินวงเงิน → Checkout ไม่ได้ (Save Draft ได้)
  • Role พิเศษ: แก้ Discount เพิ่มได้ + ปลด Credit Limit (เพิ่มชั่วคราว/ครั้ง)

Pain Points ที่ระบุ

  • ลูกค้าสั่งซ้ำเขียนกระดาษถ่ายรูปมาให้ → Template/Basket
  • ลูกค้าไม่สั่งซื้อ 1-2 เดือน → ต้องมีรายงาน
  • Search ไม่เจอ → Recommend Keyword
  • ไม่เห็นประวัติแก้ไข SO → เก็บ Log
  • ไม่เห็นสถานะเครดิต → แจ้งเตือนลูกค้าเสี่ยง

5 MoM 13 กุมภาพันธ์ 2569 — LINE / Walk-in / Pre-order / Points

13 ก.พ. 2569 — นอก Decision Register

หัวข้อ: LINE prototype, E2E Process Flow (Walk-in, CS/PC), Transfer request, Pre-order, Customer information

LINE Prototype

  • 2 หน้าจอ — กรณียังไม่ผูกเบอร์สมาชิก + กรณีผูกเบอร์สมาชิกแล้ว
  • พี่หมวยอนุมัติให้ดำเนินการต่อ

Walk-in Purchase Flow (POS)

  • Pick / Pack รวมใน Step เดียว — คนเดียวทำทั้งหมด
  • ก่อน Print ใบเตรียม → เพิ่มหน้าจอ Key Packaging (เลือกกล่อง / ถุง)
  • ไม่มี Process Packing แยกแล้ว
  • สติ๊กเกอร์ใบเตรียม ไม่ต้องมีใบปะหน้า
  • Use case: ลูกค้าสั่งล่วงหน้า + มาเลือกซื้อเพิ่ม → ชำระทีเดียว + นำของกลับไปทั้งหมด

CS / PC Flow

  • ปริ้นใบปะหน้า: สติ๊กเกอร์ + ใบเตรียมสินค้า + ถุงแดง 1 ห่อใหญ่ 1 พาเลต
  • แสดง Barcode SO + QR Code → สแกนแล้วบอกได้ว่ามีสินค้ากี่ชิ้น
  • "จ่ายเงินแล้ว" → เพิ่มเป็น "จ่ายเงินแล้ว/ตัดวงเงิน"
  • CS กับ PC ต่างกัน: PC ต้องเช็ค Shelf ของตนเองผ่าน Excel File
  • PC เช็ค Shelf ทุก 2 ชั่วโมง
  • หน้าบ้าน PC ต้องระบุความเร่งด่วน Flag ได้

Transfer Request

  • กรณีลูกค้าซื้อคนละ Flow + อยากรวมบิลเพื่อได้ Campaign
  • PC ตัดสินใจว่าจะรวมที่ไหน
  • หน้าร้านสินค้าไม่พอ → Transfer Stock จากคลังหลังบ้านมาหน้าบ้าน
  • Submit จากหน้าร้านไปคลังหลังบ้าน → Noti → ตัด Stock คลังหลังบ้าน → เพิ่ม Stock หน้าบ้าน

Pre-order NEW DETAILS

  • ลูกค้าทำได้โดย CS, CC และ Web (CS ก็ทำได้)
  • Pre-order ผูกกับคลังพิเศษ — กรณี: (1) สินค้าพิเศษ Pre-order เท่านั้น (2) สินค้าปกติของไม่พอ
  • ชำระ 100% ทุกช่องทาง
  • ช่องทางชำระ: CS/CC → Cashier / Web → Online
  • วิธีชำระ: เงินสด / Bank Transfer / Credit Line
  • 2 แบบ: (1) มีราคาจากระบบ (2) ไม่มีราคา → ติดต่อเจ้าหน้าที่
  • Pre-order ไม่ร่วม Campaign แต่ได้ Point ตาม SO
  • Quotation ไม่เช็ค Stock + ไม่คำนวน Campaign

Points (CRM) สำคัญ

  • จ่าย Point ทุกสิ้นเดือน (ไม่ใช่ทันทีที่ออก Invoice)
  • ระหว่างเดือน — ลูกค้ายังไม่เห็น Point
  • คำนวน Point รายเดือน โดยเอาใบสั่งซื้อมาหักลบจากการลดหนี้ (ลดหนี้จาก API CONX)
  • แสดงข้อความบนหน้าจอ ว่าจะคำนวน Point วันที่เท่าไหร่
  • แสดงประวัติ Point รายเดือน ย้อนหลัง 12 เดือน
  • คะแนนใช้ได้ทั้งสองปี + แสดง Point ใกล้หมด

6 MoM 27 กุมภาพันธ์ 2569 — Architecture & Website Design

27 ก.พ. 2569 — Decision Records D-021 ถึง D-025

หัวข้อ: Overall project status, E2E customer journey, High Level Architecture, YSC website

High Level Architecture

  • Cloud Platform: Google Cloud Platform (GCP) + Cloud Run (ไม่ใช้ VM)
  • Front-end 6 Instances: UI-Web, UI-Backoffice, UI-POS, UI-CRM, UI-Pickpack, Observability
  • Back-end 5 Instances: CoreAPI, BO BFF, CRM BFF, Pick&Pack BFF, POS BFF
  • Databases: Cloud SQL (Relational) + MongoDB (NoSQL) + Redis (Cache)
  • External Integration: ConX (ERP), LINE OA, IPBX (Phone)
  • Marketplace (Future): Shopee, TikTok
  • Reporting: Data Studio (Looker Studio) + Data Warehouse (Future)
  • Primary & Standby (Replica): สำหรับ Recovery

Website Design — หน้า Home

  • Promotion Banner: 4 การ์ด — ตัวอักษรใหญ่ขึ้น
  • Product Card: ชื่อสินค้า + SKU ID + Barcode
  • Member Pricing: แสดงราคาตาม Tier (P2, P4, P5) อัตโนมัติเมื่อ Login
  • Variants: ขายแบบห่อ / แบบม้วน — รหัสบาร์โค้ดและราคาต่างกัน
  • UI สำหรับผู้สูงอายุ: ตัวอักษรใหญ่ขึ้น โทนสีซอฟต์ลง
  • Icon 2 ประเภท:
    • Property Icons: Green Product, Oversize, Fragile — ตั้งที่ Master Data
    • Promotion Icons: ตามแคมเปญ ในช่วงเวลาที่กำหนด
  • Priority: Promotion Icon > Property Icon — แสดงสูงสุด 2 ไอคอน/การ์ด

Website Design — หน้ารายละเอียดสินค้า

  • ราคารวม VAT แล้ว
  • SKU + Barcode
  • 5 ช่อง Multimedia — ช่องสุดท้ายเป็น Video ถ้ามี
  • Product Remark per SKU: ลูกค้าระบุความต้องการพิเศษ (สี / "ช่วยแพ็คแยก") — ตั้งเป็น Required Field ได้
  • Available Stock: แสดงจำนวนสินค้า ให้ลูกค้าตัดสินใจ
  • "ซื้อเลย" (Buy Now) เด่นกว่า "ขอใบเสนอราคา"
  • ปุ่ม "ให้เจ้าหน้าที่ติดต่อกลับ" สำหรับสินค้าไม่มีสต็อก / ปริมาณมาก

หน้าตะกร้า — Remarks

  • Item Remark: หมายเหตุระดับสินค้า (เช่น "ช่วยแพ็คแยกออกต่างหาก")
  • Order/Delivery Remark: หมายเหตุระดับคำสั่งซื้อ ("ช่วยใส่ถุงเล็กๆ มาให้ด้วย")
  • วัตถุประสงค์: ลดการติดต่อกลับ — ลูกค้าระบุไว้ล่วงหน้า
  • Upsell: "ถ้าซื้อครบ 10,000 บาท จะได้ราคาพิเศษ"

หน้าชำระเงิน

  • Self-Pickup Time Slot: 09:00-11:00, 13:00-15:00, 15:00-17:00 — บวก 12 ชั่วโมงทำการล่วงหน้า
  • Carrier Selection: 200+ แห่ง — ตามรหัสอำเภอ/จังหวัด — Pin ที่อยู่อัตโนมัติ
  • Express Shipping: มีค่าธรรมเนียมพิเศษ
  • Payment Logic: Credit Limit แสดงเฉพาะสมาชิกที่มีสิทธิ์ + วงเงินเหลือ
  • Coupon Stacking: ใช้หลายใบพร้อมกันได้
  • e-Tax Invoice: ส่งทาง Email — ระบุแยกจากโปรไฟล์ได้
  • Quotation: PDF + ล็อคราคา 7 วัน

  • ค้นหาสินค้า — เพิ่มลงตะกร้าได้เลย
  • โปรโมชั่นแสดงในสรุปบิล + ระหว่างรายการ
  • ของแถม — พนักงาน POS หยิบมาสแกนเอง

Pick / Pack

  • Picking: เรียงตามเวลานัดรับ — ด่วนขึ้นก่อน
  • ระบบ List งาน + พนักงานกดรับงานเอง
  • หน้าจอแสดงสถานะของงานและพนักงาน
  • สินค้าไม่ครบ → แจ้ง Supervisor ดำเนินการต่อ
  • Packing: ไม่รับรถเข็น แต่ตรวจสอบรถเข็นแต่ละ SO
  • สแกนรถเข็นทั้งหมดก่อนเริ่ม + สแกนสินค้าก่อนลงบรรจุ
  • เพิ่ม UI ระบุประเภทบรรจุภัณฑ์และจำนวน
  • ปุ่มพิมพ์เอกสารใบเตรียม + สติ๊กเกอร์
  • Pack เสร็จ → ส่งข้อมูลออเดอร์ไป ConX

Logistics

9 ประเด็นสำคัญข้ามการประชุม (Cross-cutting Themes)

1. โครงสร้างคลังสินค้า 5 ประเภท

คลังใช้สำหรับ
คลังออนไลน์ (ชั้น 2, 3, 4)Web + Agent — Picker หยิบ
คลังหน้าร้าน (ชั้น 1, ชั้นลอย)POS, Walk-in — แคชเชียร์หยิบเอง
คลังพักรอ (Staging)หลัง Pack เสร็จ รอ Cashier
คลังพิเศษ (Pre-order)Pre-order — สต็อกติดลบได้
คลังของรางวัล (Reward)การแลกคะแนน — ไม่ใช่ของแถม

2. ช่องทางการขาย 3 ช่องทาง + Sub-channels

  • Web: ลูกค้า B2C/B2B สั่งเอง — บัตร / โอน / เครดิต
  • Agent (CC, CS, Sales): Staff สั่งแทนลูกค้า
  • POS: หน้าร้าน — ชำระหน้าร้าน
  • Shopee และ Social — CC นำเข้าระบบเอง (รอเฟส 2)

3. ระดับสมาชิก (Member Tier) 7 ระดับ

  • Prospect / Member / Silver / Gold / Platinum / Diamond / Coronet
  • Prospect → Member: ยอดในคำสั่งซื้อใบเดียว ≥ 10,000 (Web) หรือ 4,000 (POS)
  • Silver ขึ้นไป: ยอดสะสมปีปฏิทิน 100K / 300K / 1M / 3M

4. Price Tier (P1-P5)

  • Ecom Default = P5 / CC, POS Default = P4 (ปรับได้)
  • Ecom Order ≥ 10K → P2 ทันที
  • POS Order ≥ 4K → P2 ทันที

5. Points Logic

  • 1 Point = 1,500 บาท (PT-01)
  • Prospect ไม่ได้รับ Point
  • จ่ายทุกสิ้นเดือน (ไม่ใช่ทันทีหลัง Invoice)
  • คำนวณรายเดือน หลังหักลดหนี้จาก ConX
  • แสดงประวัติย้อนหลัง 12 เดือน
  • หมดอายุ 31 ธันวาคมทุกปี

6. Pre-order

  • ช่องทาง: CS / CC / Web
  • ชำระ 100% ทุกช่องทาง
  • ไม่ร่วม Campaign — ได้ Point ตาม SO
  • คลังพิเศษ — สต็อกติดลบได้
