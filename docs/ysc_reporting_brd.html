2. วัตถุประสงค์เชิงธุรกิจ

ระบบรายงานเป็นเครื่องมือของผู้บริหารและหัวหน้างานในการดูข้อมูลและตัดสินใจ ครอบคลุมข้อมูลจากทุกโมดูล (OMS, CRM, PRD, WMS, Promotion) แสดงเป็นแดชบอร์ดและรายงานเชิงปฏิบัติการ พร้อมการส่งออกและแจ้งเตือนเมื่อตัวเลขเข้าเกณฑ์ที่กำหนด

รหัสเป้าหมายธุรกิจตัวชี้วัด
BG-RPT-1ผู้บริหารเห็นภาพรวมธุรกิจในที่เดียวมีแดชบอร์ดที่เห็นยอดขาย, สต็อก, ลูกค้า, แคมเปญ ครบ
BG-RPT-2หัวหน้างานเข้าถึงรายงานที่ต้องการเฉพาะส่วนรายงานครบทุกโมดูล + RBAC
BG-RPT-3ลดเวลาการทำรายงานรายเดือนเวลาทำรายงาน ลดลง 80%
BG-RPT-4แจ้งเตือนทันทีเมื่อมีปัญหาแจ้งเตือนภายใน 5 นาทีจากเหตุการณ์

3. ขอบเขตของงาน

3.1 อยู่ในขอบเขต (In Scope)

  • แดชบอร์ดผู้บริหาร (Executive Dashboard)
  • รายงานเชิงปฏิบัติการครบทุกโมดูล: OMS, CRM, PRD, WMS, Promotion, POS
  • การกรองข้อมูลตามช่วงเวลา / ช่องทาง / สาขา / สินค้า
  • การส่งออกเป็น Excel / PDF / CSV
  • การตั้งการแจ้งเตือนเมื่อค่าเข้าเกณฑ์
  • การสร้างรายงานกำหนดเอง (Custom Report)
  • การส่งรายงานอัตโนมัติทางอีเมลตามกำหนด
  • การจัดการสิทธิ์การเข้าถึงรายงาน

3.2 ไม่อยู่ในขอบเขต (Out of Scope)

  • รายงานทางบัญชี / ภาษีจาก ConX (ใช้ระบบ ConX เอง)
  • รายงานวิเคราะห์เชิงลึก (Predictive Analytics) ทำในเฟสต่อไป

4. ผู้มีส่วนได้ส่วนเสีย

บทบาทหน้าที่สิทธิ์
ผู้บริหารดูแดชบอร์ดและรายงานภาพรวมทั่วทั้งระบบ
หัวหน้างานแต่ละฝ่ายดูรายงานเฉพาะฝ่ายของตนเฉพาะโมดูล
ฝ่ายการตลาดดูรายงานแคมเปญและพฤติกรรมลูกค้าการตลาด
ฝ่ายการเงินดูรายงานวงเงินเครดิตและยอดค้างชำระการเงิน
หัวหน้าคลังดูรายงานสต็อกและประสิทธิภาพคลังคลัง

5. สถานะปัจจุบัน (As-Is)

  • รายงานทำโดยทีม IT ฝ่ายต่างๆ ต้องร้องขอ
  • ใช้ Excel ด้วยมือ ดึงข้อมูลจาก ConX แล้วทำ Pivot Table
  • ไม่มีแดชบอร์ดเวลาจริง ดูเป็นรายงานสรุปทุกสิ้นเดือน
  • ไม่มีการแจ้งเตือนอัตโนมัติ

6. กระบวนการในอนาคต (To-Be)

  • แดชบอร์ดเวลาจริง ทุกฝ่ายเข้าถึงเอง
  • รายงานสำเร็จรูป ครอบคลุมทุกโมดูล
  • การส่งออกในรูปแบบต่างๆ Excel / PDF / CSV
  • การแจ้งเตือนเมื่อค่าผิดปกติ
  • การส่งรายงานอัตโนมัติ รายวัน / สัปดาห์ / เดือน ตามที่ผู้ใช้ต้องการ

SCR-RPT-004หน้าจอรายงานลูกค้าลูกค้าใหม่ การเลื่อน/ลดระดับ คะแนน
SCR-RPT-005หน้าจอรายงานแคมเปญประสิทธิภาพและ ROI
SCR-RPT-006หน้าจอรายงาน CCประสิทธิภาพ Contact Center
SCR-RPT-007หน้าจอสร้างรายงานกำหนดเองสร้างรายงานตามต้องการ
SCR-RPT-008หน้าจอตั้งแจ้งเตือนกำหนดเงื่อนไขและช่องทางการแจ้ง

9. กฎเกณฑ์เชิงธุรกิจ

NTF-06การแจ้งเตือนหัวหน้างาน (Supervisor Alert)
-การคำนวณ KPI ทุกตัวต้องสอดคล้องกับกฎที่กำหนดในแต่ละโมดูล

10. การเชื่อมต่อระบบภายนอก

ระบบข้อมูลทิศทาง
OMS / CRM / PRD / WMS / Promotionข้อมูลปฏิบัติการสำหรับสร้างรายงานโมดูล → Reporting
ConX ERPข้อมูลทางบัญชีและการเงินConX → Reporting
ระบบส่งอีเมล / LINEการส่งรายงานอัตโนมัติและการแจ้งเตือนReporting → ผู้ใช้

11. การออกแบบข้อมูลและข้อมูลหลัก

หน่วยข้อมูลคำอธิบายคีย์หลัก
Report Templateเทมเพลตรายงานสำเร็จรูปTemplate ID
Custom Reportรายงานที่ผู้ใช้สร้างเองReport ID
Alert Ruleกฎและเงื่อนไขการแจ้งเตือนAlert ID
Scheduleการส่งรายงานอัตโนมัติตามกำหนดSchedule ID
Data Warehouseข้อมูลรวมจากทุกโมดูลสำหรับรายงาน(หลายตาราง)

12. ความปลอดภัยและสิทธิ์การเข้าถึง

บทบาทเข้าถึงรายงาน
ผู้บริหารทุกโมดูล + แดชบอร์ดผู้บริหาร
หัวหน้างานแต่ละฝ่ายเฉพาะโมดูลของตน
พนักงานปฏิบัติการเฉพาะรายงานที่ได้รับสิทธิ์

13. ข้อกำหนดเชิงคุณภาพ (NFR)

หัวข้อเกณฑ์
เวลาตอบสนองโหลดรายงานปกติ ≤ 5 วินาที / รายงานใหญ่ ≤ 30 วินาที
ความถี่อัปเดตแดชบอร์ดรีเฟรชทุก 15 นาที / รายงานปฏิบัติการเป็นเวลาจริง
การส่งออกExcel ≤ 100,000 แถว / PDF ≤ 1,000 หน้า
การเก็บข้อมูลย้อนหลัง ≥ 3 ปี

14. รายงานและแดชบอร์ดที่ระบบรองรับ

หมวดรายงาน
ยอดขายรายงานยอดขายรายวัน/สัปดาห์/เดือน, Top สินค้า, Top ลูกค้า, ยอดตามช่องทาง/สาขา
คำสั่งซื้อคำสั่งซื้อตามสถานะ, Auto-cancel, การคืน/CN-DN, สลิปรอตรวจ
สินค้าสินค้าที่เปลี่ยนสถานะ, ราคาที่จะเปลี่ยน, สินค้าไม่เคลื่อนไหว, สีสถานะคลัง 4 สี
คลังสินค้าสต็อกคงเหลือ, การเคลื่อนไหว, ประสิทธิภาพ Pick/Pack, การตรวจนับ
ลูกค้าลูกค้าใหม่, การเลื่อน/ลดระดับ, คะแนนสะสม, วงเงินเครดิตที่ใช้
แคมเปญประสิทธิภาพแคมเปญ, ROI, การใช้คูปอง, Cost of Discount
Contact Centerคำสั่งซื้อต่อพนักงาน, เวลาให้บริการ, อัตรา Closure
POS / หน้าร้านยอดขาย POS ต่อสาขา/แคชเชียร์, การยกเลิก, การคืน

15. ข้อกำหนดการโอนย้ายข้อมูล

  • ข้อมูลย้อนหลังจากระบบเก่าและ ConX โอนเข้า Data Warehouse
  • ตรวจสอบความถูกต้องของยอดขายและสต็อกย้อนหลัง ≥ 3 ปี

16. สมมติฐานและข้อจำกัด

  • มี Data Warehouse แยกจาก Production เพื่อไม่กระทบประสิทธิภาพ
  • การเปรียบเทียบกับข้อมูลก่อน Go-live อาจไม่สมบูรณ์
