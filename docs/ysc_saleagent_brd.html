2. วัตถุประสงค์เชิงธุรกิจ

ช่องทางการขายผ่านพนักงาน (Sale Agent) เป็นช่องทางที่พนักงานของบริษัทสร้างคำสั่งซื้อในระบบแทนลูกค้า ครอบคลุม 3 กลุ่มพนักงาน ได้แก่

  • CC — Contact Center: รับออเดอร์ผ่านโทรศัพท์, LINE OA, อีเมล หรือ Fax ป้อนข้อมูลเข้าระบบแทนลูกค้า
  • CS — Customer Service: ดูแลปัญหาหลังการขาย รับคำขอเปลี่ยน/แก้ไขคำสั่งซื้อ ติดตามสถานะแทนลูกค้า
  • PC — Product Consultant: แนะนำสินค้า รับออเดอร์จากลูกค้าที่เดินเข้ามาหรือโทรปรึกษา สร้างคำสั่งซื้อผ่านระบบหลังบ้านโดยตรง

วัตถุประสงค์หลัก

  • ลดขั้นตอนการทำงานด้วยกระดาษ (เดิม: พิมพ์ใบ → นำไปคีย์ → เกิดข้อผิดพลาด) เป็นระบบดิจิทัลครบวงจร
  • เพิ่มโอกาส Up-sell / Cross-sell ด้วยข้อมูล Top 10 สินค้าที่ลูกค้าซื้อประจำและการแจ้งเตือนโปรโมชัน
  • ให้พนักงานเห็นข้อมูลลูกค้าครบถ้วน (ระดับสมาชิก, ยอดซื้อ, วงเงินเครดิต) บนหน้าจอเดียวก่อนสร้างคำสั่งซื้อ
  • รองรับการบันทึก Sub-channel การสั่งซื้อ (โทรศัพท์ / LINE / อีเมล / เดินเข้ามา) เพื่อการรายงานและวิเคราะห์ช่องทาง
  • รองรับการทำงาน 2 จอพร้อมกัน: จอสนทนากับลูกค้า + จอสร้างคำสั่งซื้อ

3. ขอบเขตของงาน

อยู่ในขอบเขต (In Scope)
  • Agent Portal: หน้าจอรวมข้อมูลลูกค้า + สร้างคำสั่งซื้อ
  • สร้างคำสั่งซื้อปกติ / Pre-order / ใบเสนอราคาผ่าน Backoffice
  • แปลงใบเสนอราคาเป็นคำสั่งซื้อ
  • ยกเลิกและสร้างคำสั่งซื้อทดแทน (Cancel + Duplicate)
  • ส่งข้อมูลชำระเงิน (QR Code / บัญชีโอนเงิน) ให้ลูกค้าผ่านระบบ
  • ติดตามคำสั่งซื้อรอชำระเงิน (SO Monitoring — 24 ชม.)
  • ขออนุมัติวงเงินชั่วคราว (Over-limit Approval)
  • บันทึก Sub-channel การสั่งซื้อ (feature วางแผนไว้)
  • การแจ้งเตือน Up-sell / Cross-sell
  • การใช้คูปองแทนลูกค้าในขั้นตอน Checkout
อยู่นอกขอบเขต (Out of Scope)
  • การสร้างคำสั่งซื้อผ่านเว็บไซต์หน้าร้านออนไลน์ (ดู ysc_storefront_brd.html)
  • การขาย ณ จุดขายหน้าร้าน POS (ดู ysc_pos_brd.html)
  • การออก Invoice / Cashier Gate (ดู ysc_oms_brd.html)
  • การจัดการสต็อกและ Pick/Pack (ดู ysc_wms_brd.html)
  • Ticket System / After-sales Service (อยู่ใน CRM)
  • การจัดการโปรโมชันและแคมเปญ (ดู ysc_promotion_brd.html)
  • LINE OA Chatbot / Automated Messaging (ไม่ใช่ขอบเขต Adeptio)
  • Shopee / Social Commerce integration อัตโนมัติ (CH-05 — CC นำเข้าเองด้วยตนเอง)

4. ผู้มีส่วนได้ส่วนเสียและบทบาทผู้ใช้งาน

บทบาทชื่อย่อความรับผิดชอบหลักสิทธิ์ในระบบ
พนักงาน Contact Center CC รับออเดอร์ทางโทรศัพท์ / LINE / อีเมล / Fax, สร้างตะกร้า, ส่งข้อมูลยืนยันให้ลูกค้า, ติดตาม SO รอชำระ สร้าง/อ่าน SO, Quotation, Pre-order; ขอ Over-limit Approval; ใช้คูปองแทนลูกค้า
พนักงาน Customer Service CS แก้ไขคำสั่งซื้อ (Cancel + Duplicate), ติดตามสถานะแทนลูกค้า, รับ Complaint สร้าง/อ่าน SO, Cancel SO ที่ยังไม่ Pick; ดู Order History

ผู้จัดการ Contact Center CC Supervisor อนุมัติ Over-limit วงเงินชั่วคราว, ดูรายงาน SO Monitoring อนุมัติ Over-limit (ตามวงเงิน Approval Matrix); ดู Report

จุดประสงค์

พนักงาน CC / CS / PC เปิดหน้าจอ Agent Portal แล้วระบบดึงข้อมูลลูกค้าขึ้นแสดงทันทีเมื่อรับสายหรือค้นหาด้วยเบอร์โทรศัพท์ เพื่อให้พนักงานเห็นบริบทลูกค้าครบก่อนสนทนา

เงื่อนไขก่อนเริ่มต้น
  • พนักงานล็อกอินเข้าระบบ Backoffice ด้วยบัญชีที่มีสิทธิ์ CC / CS / PC
  • หน้าจอ Customer Connect (มัลติเอเจนต์) เปิดอยู่
เหตุการณ์เริ่มต้น

พนักงานรับสายโทรศัพท์หรือเริ่มแชทกับลูกค้า — ระบบได้รับเบอร์โทรศัพท์ หรือพนักงานกรอกเบอร์ / ชื่อลูกค้าเพื่อค้นหา

ขั้นตอนหลัก
  1. ระบบรับเบอร์โทรศัพท์ของลูกค้า (จาก CTI หรือพนักงานกรอกเอง)

16. สมมติฐานและข้อจำกัด

  • พนักงาน CC / CS / PC ทุกคนมีบัญชีผู้ใช้งานใน Backoffice และกำหนด Role ไว้แล้ว
  • ระบบ Customer Connect (Multi-Agent) รองรับการทำงานพร้อมกันหลายพนักงาน
  • CTI (Computer Telephony Integration) ที่ดึงเบอร์โทรศัพท์เข้าหน้าจออัตโนมัติ — อยู่นอกขอบเขตของ Adeptio; หากไม่มี พนักงานกรอกเองด้วยตนเอง
  • LINE OA ใช้เป็นช่องทางสื่อสาร — ไม่ใช่ช่องทางสร้าง Order โดยตรง (CC สร้าง Order ใน Backoffice แทน)
  • Approval Matrix สำหรับ Over-limit [รอยืนยันจาก YSC — P-02]

17. เกณฑ์การยอมรับผลงาน (Module-level)

AC-SA-MOD-01: พนักงาน CC / CS / PC สามารถสร้าง SO ได้ครบทุก Flow (Normal / Pre-order / Quotation) ผ่าน Backoffice โดยไม่ต้องใช้กระดาษ

AC-SA-MOD-05: Cancel + Duplicate ทำงานถูกต้อง — SO ใหม่อ้างอิง SO ต้นทาง, สต็อกและวงเงินคืนทันที
AC-SA-MOD-06 [SIT]: SO Backoffice ส่งข้อมูลไปยัง ConX ERP สำเร็จ ภายใน 10 วินาทีหลัง Confirm
